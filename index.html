vector.js lets you apply standard +-/* arithmetic operators in JavaScript, to vectors just as you would with simple numbers.
The project serves as a template to demonstrate how to emulate overloading of JavaScript operators. These operators can be applied to your own JavaScript objects.

Ad hoc

# Embed the file lsauer/vector-js/vector-op.js
    <script type="text/javascript" src="http://lsauer.github.com/vector-js/vector-op.js"></script>

  • Create a new vector

        a=new vectorop.Vector(4, 10, 100)
    +a   //invoke a.toString();
    //> [x: 4, y: 10, z: 100]
    //> 6556164

  • Perform basic vector arithmetics

        if (typeof Vector !== 'undefined'){
    	throw new Error("Vector object already exists in current scope!")
    }else with(vectorop){    
        var a = Vector(0, 3, 0);
        var b = Vector(2, 0, 1);
        var c = Vector(1, 0, 1);
        var d = Vector(a + b - c);
        +d
        //> [x: 1, y: 3, z: 0]
    }
    
    var d = new vectorop.Vector(1, 2, 3);
    var scalarf = 2.3;
    var e = vectorop.Vector(d * scalarf);
    delete e; // delete the vector instance 'e'
    
    var a = new vectorop.Vector(10, 20, 30);
    var b = new vectorop.Vector(a/10);
    +b
    //> [x: 1, y: 2, z: 3]

Introduction

Operators in JavaScript actually correspond to native JavaScript classes, yet overloading proposals were rejected due to warranted browser security concerns. The Addition operator for instance internally calls a prototype['+'] method.


In the case of a Number object the function Number.prototype['+'] is invoked, to perform number-addition, and in the case of a string, String.prototype['+'] performs string concatenation. Adobe ExtendScript actually allows straightforward operator overloading in this described manner. JavaScript does not.

Available still, is the capture and manipulation of implicit type coercion invocation via valueOf and stringTo. ,Implicitly for instance during the == operator, and explicitly , for instance during the use of the +-operator,

In order to emulate operator overloading with complex (JSON) objects, information has to be bit-serialized or packed, and conversly, unpacked to retain full JavaScript objects. This is demonstrated in vector.js as a boiler-plate, ready to be adapted to individual operator-usage needs.

For instance, a typical scenario where emulated operator overloading is helpfu,l are JavaScript Read–eval–print loops, or REPLs for short.

Usage

Given a Vector instance myvector, created via myvector = vectorop.Vector(1,2,3);, you can:
Use the myvector.length attribute to get the vector's magnitude

myvector.unit() Calculates the unit vector
myvector.div(v2)
myvector.mul(v2)		 Divides, respectively multiplies the vector instance with a vector instance passed as argument
myvector.dot(v2) Calculates the dot product
myvector.cross(v2) Calculates the cross product
myvector.dotcross(v2,v3) Calculates the scalar triple product
myvector.toString(true) Provides a pretty print output of the vector instance

Use the r,p,t attributes of the myvector.spherical() function to get spherical coordinates in radians

myvector.spherical() Returns the the magnitude r, phi p and theta t of the vector instance.
myvector.spherical(r,p,t) Sets the vector instance to the given spherical values.

See the Code-section for more information about all functions.
